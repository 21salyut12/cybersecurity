# Man-in-the-Middle Attack  

## Main Objectives
The scope of this project/experiment is to raise awareness about this type of cyber attack and about how vulnerable can ARP if not protected correctly.

## What is MitM
MitM also known as Man-in-the-Middle is a cyber attack where a threat actor eavesdrops on the communication between certain devices and eventually manipulates it in such a way that benefits their malicious intent without either end of the communication knowing it.

## Description  
This project consists of a simulated Man-In-The-Middle (MITM) attack using Kali Linux and GNS3. The attack was carried out through ARP spoofing, allowing the attacker to intercept and manipulate traffic between two virtual PCs.

## Conclusions

### Objective
The experiment demonstrated the effectiveness of ARP spoofing in compromising network security. The attack enabled traffic interception and modification, highlighting the risks of unprotected network environments.  

## Mitigation Techniques  
To prevent MITM attacks, the following security measures can be implemented:  
1. Dynamic ARP Inspection (DAI) to block spoofed ARP packets.  
2. Static ARP entries for critical devices to prevent ARP poisoning.  
3. VLAN segmentation to limit the impact of spoofing within a network.  
4. Encryption protocols (HTTPS, TLS, VPNs) to secure communication even if traffic is intercepted.
